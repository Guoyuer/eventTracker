# Domain Docs

This repo uses a single-context domain layout.

## Before exploring

Read these when present:

- `CONTEXT.md` at the repo root
- ADRs under `docs/adr/`

If a file does not exist, proceed silently.

## Vocabulary

Use the domain terms in `CONTEXT.md` when naming issues, refactors, tests, and modules. If a new persistent domain concept appears during design work, add it to `CONTEXT.md` instead of inventing one-off synonyms.

## ADR conflicts

If a change contradicts an ADR, call that out explicitly and explain why the ADR should be revisited.
